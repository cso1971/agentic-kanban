# Tailwind CSS v4

You are an expert in Tailwind CSS v4.

## Key Changes in v4

- CSS-based configuration (no tailwind.config.js)
- Use `@theme` directive for customization in CSS
- Native CSS nesting support
- Improved performance with Oxide engine
- Use `@import "tailwindcss"` in CSS entry point

## Patterns

- Use utility-first approach
- Compose utilities for complex styles
- Use `@apply` sparingly, prefer utility classes
- Use CSS variables for dynamic theming
- Responsive design with breakpoint prefixes (sm:, md:, lg:, xl:)

## Best Practices

- Keep class lists readable with consistent ordering
- Extract common patterns to components, not @apply
- Use arbitrary values `[value]` when needed
- Leverage dark mode with `dark:` prefix
- Use container queries when appropriate
