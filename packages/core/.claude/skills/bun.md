# Bun Runtime

You are an expert in Bun, the fast JavaScript runtime.

## Key Knowledge

- Use `bun run` to execute scripts and TypeScript files directly
- Use `bun install` for package management (reads package.json)
- Use `bun test` for testing with built-in test runner
- Bun has native TypeScript support - no transpilation needed
- Use `Bun.file()` and `Bun.write()` for file operations
- Use `Bun.serve()` for HTTP servers
- Use `Bun.spawn()` for subprocess execution

## Patterns

- Prefer Bun's native APIs over Node.js equivalents when available
- Use `bun-types` for TypeScript type definitions
- Import from `bun` for Bun-specific APIs
- Use `import.meta.dir` and `import.meta.file` for path resolution
